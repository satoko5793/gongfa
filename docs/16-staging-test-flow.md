# 16 测试服测试流程

更新时间：2026-03-17

这份文档解决一个问题：

- 每次代码改完并发布到测试服后，按固定流程判断这次修改有没有把主流程带坏。

当前测试服默认通过 SSH 隧道访问：

```bash
ssh -L 8081:127.0.0.1:8081 root@101.34.247.186
```

然后在本机打开：

- `http://127.0.0.1:8081/`
- `http://127.0.0.1:8081/admin.html`
- `http://127.0.0.1:8081/health`

## 先判断跑哪套流程

跑小测试：

- 只改了文案、样式、静态页面结构
- 只改了前端展示，不改下单、充值、登录、后台审核逻辑
- 只改了后台表单布局、筛选、列表展示
- 只修了明显前端 bug，影响范围可控

跑大测试：

- 改了登录、注册、用户资料、密码
- 改了商品查询、商品导入、库存、价格、上下架
- 改了下单、取消申请、后台审核
- 改了充值、赛季会员、额度流水、审计日志
- 改了后端路由、校验、配置、数据结构、部署文件
- 不确定影响范围时，默认跑大测试

## 通用前置步骤

1. 本地代码先提交或至少保证当前改动清晰。
2. 发布到测试服。
3. 确认测试服健康检查正常：

```bash
curl http://127.0.0.1:8081/health
```

4. 先跑一次前端校验：

```bash
python scripts/validate_frontend.py
```

5. 如果这次主要是页面或静态资源改动，再人工强刷浏览器缓存。

## 小测试流程

适用目标：

- 快速确认“这次小改动没把页面主入口搞坏”。

### Step 1. 基础可用性

- 打开首页，确认页面能加载，不是白屏
- 打开后台页，确认页面能加载，不是白屏
- 首页商品列表能显示
- 后台主要卡片能显示

### Step 2. 改动点回归

- 逐个检查这次实际修改的页面、弹层、按钮、文案
- 如果改了样式，桌面端和移动端宽度都看一遍
- 如果改了筛选、排序、搜索，至少手动点一遍

### Step 3. 主链路冒烟

- 用户登录一次
- 后台管理员登录一次
- 打开一个商品详情弹层
- 打开“我的账户”区域

### Step 4. 通过标准

满足下面四条就算小测试通过：

- 首页正常
- 后台正常
- 本次改动点表现正确
- 登录和商品详情没有被带坏

## 大测试流程

适用目标：

- 确认核心业务链路没有回归。

### Step 1. 基础检查

- 首页可访问
- 后台可访问
- `/health` 正常
- 商品列表有数据
- 浏览器控制台没有明显报错

### Step 2. 用户与权限

- 新用户注册或用测试账号登录
- 修改一次资料
- 修改一次密码
- 确认未登录时受保护操作会被拦截
- 管理员登录后台

### Step 3. 商品与展示

- 首页商品列表正常
- 搜索可用
- 排序可用
- 商品详情字段完整
- 库存和价格展示正常

### Step 4. 额度与购买

- 管理员给测试账号加额度
- 用户刷新后看到额度变化
- 用户下单一个有库存商品
- 订单记录能看到新订单
- 额度扣减正确

### Step 5. 取消与审核

- 用户对刚创建的订单发起取消申请
- 后台能看到取消申请
- 管理员审核取消
- 用户额度正确回退
- 订单状态更新正确

### Step 6. 充值与会员

- 用户打开充值面板
- 确认当前充值比例、最低金额、二维码显示正常
- 提交一笔普通充值申请
- 后台能看到充值单
- 管理员审核通过
- 用户额度到账正确
- 如果本次改动涉及赛季会员，再补测一笔赛季会员申请与审核

### Step 7. 后台运营能力

- 商品导入入口能打开
- 用户搜索可用
- 订单查询可用
- 充值配置可读可保存
- 额度流水能查
- 审计日志能查

### Step 8. 发布前结论

大测试通过，至少要满足：

- 登录正常
- 商品正常
- 下单正常
- 取消审核正常
- 充值审核正常
- 后台关键页面正常

## 推荐耗时

- 小测试：5 到 10 分钟
- 大测试：20 到 40 分钟

## 推荐记录格式

每次测完至少记这几项：

- 测试时间
- 测试人
- 测试服版本或提交号
- 跑的是小测试还是大测试
- 结果：通过 / 不通过
- 发现的问题

示例：

```text
时间：2026-03-17 18:30
提交：64f8703
环境：staging
流程：大测试
结果：通过
备注：充值和取消链路正常，后台审计日志正常
```

## 什么时候不能直接发正式

出现下面任意一种，不要发正式：

- 首页白屏
- 后台白屏
- 登录失败
- 下单失败
- 充值审核结果不对
- 额度扣减或回退不对
- 控制台或后端日志出现明显报错但原因未查清
