# 测试服前端与 Helper 冒烟测试标准

## 目标

这份文档用于统一测试服的最小可用验证流程，优先覆盖：

1. 注册与密码登录是否可用
2. 登录后账号信息是否正常展示
3. helper 扫码绑定入口是否可打开
4. 正式服与测试服 helper 是否完全隔离
5. 前端主要按钮和页面切换是否可交互

## 测试环境

- 商城测试服：`http://127.0.0.1:8081/`
- helper 测试服：`http://127.0.0.1:8083/`
- 正式 helper：`http://127.0.0.1:8082/`

当前推荐直接使用公网测试服：

- 商城测试服：`http://101.34.247.186:8081/`
- helper 现阶段建议优先使用商城内实验入口或既有 helper 路径联调

如需本地隧道辅助排查，可使用：

```bash
ssh -L 8081:127.0.0.1:8081 -L 8082:127.0.0.1:8082 -L 8083:127.0.0.1:8083 root@101.34.247.186
```

## 测试分层

### 1. 接口冒烟

目标：

- 确认后端路由与数据源正常
- 排除“页面空白其实是接口没返回”的误判

### 2. 前端交互冒烟

目标：

- 确认主脚本已成功启动
- 确认关键按钮、tab、弹窗、列表、分页、hash 跳转可用

## 接口冒烟用例

### A. 页面基础可交互

1. 打开商城测试服首页。
2. 强制刷新一次页面，避免旧缓存干扰。
3. 检查“快速注册 / 密码登录”标签是否可以切换。
4. 检查“注册并登录 / 登录 / 退出登录”按钮点击后是否有提示文案变化。

通过标准：

- 标签可切换
- 点击按钮后页面有即时反馈，不是完全无响应

### B. 密码注册

1. 填写随机 `游戏 ID`
2. 填写 `游戏名称`
3. 填写密码与确认密码
4. 点击“注册并登录”

通过标准：

- 接口返回成功
- 页面进入 `#account`
- 概览区显示角色名、游戏 ID、登录方式、额度

### C. 密码登录

1. 使用已注册测试账号
2. 切换到“密码登录”
3. 输入 `游戏 ID` 与密码
4. 点击“登录”

通过标准：

- 接口返回成功
- 页面进入 `#account`
- 账号概览和顶部状态栏同步显示登录信息

### D. 登录态接口一致性

登录成功后，验证下列接口：

- `GET /auth/me`
- `GET /me/quota`
- `GET /me/orders`
- `GET /helper/config`

通过标准：

- 前三个接口返回 200
- `/auth/me` 与页面显示的 `游戏 ID / 名称 / 登录方式` 一致
- `/helper/config` 返回 `enabled=true` 且 `mode=staging`

### E. helper 扫码绑定入口

1. 登录商城测试账号
2. 进入“游戏助手实验区”
3. 点击“打开扫码绑定窗口”

通过标准：

- 成功打开 helper 测试页
- 地址指向 `127.0.0.1:8083`
- 不会跳到正式 helper

### F. 正式服隔离

验证：

- `8082` 和 `8083` 都健康
- 测试服商城 helper 配置指向 `8083`
- 正式商城不暴露实验入口

## 前端交互冒烟用例

### G. 页面启动与首屏渲染

1. 打开首页。
2. 强制刷新一次，避免缓存影响。
3. 检查商品区是否出现商品卡片。
4. 检查“我的账户”区域是否出现。
5. 检查右下角调试面板是否出现启动状态。

通过标准：

- 商品区不是空白
- 账户区结构完整
- 调试面板中没有明显 `window.error` / `window.rejection`

### H. 登录注册交互

验证以下控件：

- `快速注册` tab
- `密码登录` tab
- `注册并登录`
- `登录`
- `退出登录`
- `切换账号`

通过标准：

- tab 可切换
- 点击按钮后页面有反应
- 登录后进入 `#account`
- 登录后账户信息和余额可见

### I. 商品区交互

验证以下控件：

- 分类 tab
- 子分类 tab
- 搜索框
- 排序框
- `详情`
- `购买`
- `转账锁卡`
- 分页按钮

通过标准：

- 切换筛选后商品列表会变化
- 点击详情后能打开商品弹窗
- 商品按钮不会完全无响应

### J. 账户区交互

验证以下控件：

- `概览`
- `资料`
- `安全`
- `充值`
- `订单帮助`
- `保存资料`
- `更新密码`

通过标准：

- tab 可切换
- 表单提交后有成功或失败提示
- 不允许出现点击完全无响应

### K. Helper 实验区交互

验证以下控件：

- `打开 helper`
- `打开扫码绑定窗口`
- `保存当前选择`
- `清空待绑定`
- `解绑当前绑定`

通过标准：

- 实验区能展示当前配置
- helper 入口能打开
- 按钮有明确提示反馈

### L. 全局导航与底部 Dock

验证以下控件：

- 顶部导航 `商城 / 登录 / 我的`
- 底部 Dock `商城 / 我的信息 / 拍卖 / 代抽专区`
- hash 跳转

通过标准：

- 点击后页面能滚动到对应区域
- 高亮状态能随滚动或点击变化

### M. 商品弹窗交互

验证以下控件：

- `关闭`
- `确认购买`
- `转账锁卡`
- 锁卡表单的取消与提交

通过标准：

- 弹窗能打开和关闭
- 弹窗里的按钮可点击
- 提交后有反馈
