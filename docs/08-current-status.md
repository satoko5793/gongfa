# 08 当前状态总览

更新时间：2026-03-15

## 1. 当前项目是什么
项目已经从早期的“记录 / 对账”方向，收敛成“功法卡片商城 MVP”。

当前主目标是先跑通这条业务链路：

1. 管理员导入卡片 JSON
2. 后台生成并管理商品
3. 用户注册 / 登录
4. 用户浏览商品并用额度购买
5. 管理员管理用户、额度、订单、商品

当前主线仍然是网页端。
小程序端已经在并行推进，但后台管理仍以网页为主。

## 2. 当前可用入口

本地默认端口统一为 `8090`。

- 前台商城：`http://127.0.0.1:8090/`
- 后台管理：`http://127.0.0.1:8090/admin.html`
- 健康检查：`http://127.0.0.1:8090/health`
- helper 开发服务（非主流程必需）：`http://127.0.0.1:3000/`

启动方式：

```bat
start-local.cmd
```

## 3. 当前运行模式

当前默认使用“本地文件存储模式”，方便直接联调，不依赖 PostgreSQL。

本地数据文件：

- `backend/dev-data.json`

这个文件里会保存：

- 用户
- 额度账户
- 商品
- 套餐 SKU
- 订单
- 审计日志

如果后续切回 PostgreSQL，结构以 `infra/schema.sql` 为准。

## 8. 定价系统现状

当前价格统一由后端计算。

关键点：

- 网页端和小程序端读取的是同一套后端价格
- 本地文件存储支持按定价版本自动重算
- `高攻全图鉴` 当前价格已修正为 `8000`
- 金卡 / 红卡 / 橙卡 / 紫卡双满与非双满规则，已按最近确认的锚点收口

详细规则见：

- `docs/06-pricing-system.md`

## 9. 当前排序规则

前台默认排序已经与网页端一致：

1. 套餐优先，按 `display_rank`
2. 普通卡按 `legacy_id` 从大到小
3. 同类卡先双词条，再单词条，再无词条
4. 再按词条总值
5. 再按走火
6. 再按气定
7. 最后按攻击、血量

## 10. 图鉴封面规则

图鉴套餐会自动拼接封面，不再只显示文字占位。

当前规则：

- 橙卡及以下：橙 / 紫 / 蓝 / 绿
- 红及以下：红 / 橙 / 紫 / 蓝
- 满攻击全套图鉴 / 高攻全图鉴：金 / 红 / 橙 / 紫 / 蓝 / 绿

## 12. 本地联调脚本

仓库里已经补了一个最小烟雾测试脚本：

- `scripts/smoke-local-order-cancel.js`

它会自动跑完整链路：

1. 管理员登录
2. 注册临时测试用户
3. 给测试用户加额度
4. 下单购买
5. 用户提交取消申请
6. 管理员审核取消

运行方式：

```bash
node scripts/smoke-local-order-cancel.js
```

默认地址是 `http://127.0.0.1:8090`。
如果要切别的环境，可以这样跑：

```bash
GONGFA_BASE_URL=http://127.0.0.1:8090 node scripts/smoke-local-order-cancel.js
```
