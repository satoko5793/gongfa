# 网页版上线前检查清单

更新时间：2026-03-16

当前项目以网页前台和网页后台为唯一主线。下面这份清单用于上线前自检，也可作为联调完成后的验收表。

## 1. 基础运行

- [ ] `backend/` 可本地启动，`/health` 返回 `ok: true`
- [ ] `frontend/` 可直接访问并正常请求后端接口
- [ ] 环境变量已区分本地、生产，未硬编码测试地址
- [ ] 生产部署方式明确（本机 / 云主机 / 容器）

## 2. 账号与权限

- [ ] 用户可注册
- [ ] 用户可登录
- [ ] 管理员可登录后台
- [ ] 非管理员无法访问后台管理接口
- [ ] Token 失效或缺失时有明确提示

## 3. 商品与套餐

- [ ] 商品列表能正常展示
- [ ] 商品详情数据完整
- [ ] 搜索和筛选可用
- [ ] 无库存商品不会误下单
- [ ] 后台商品导入、编辑、上架流程可用

## 4. 额度与订单

- [ ] 用户额度展示正确
- [ ] 下单会扣减正确额度
- [ ] 订单列表和订单详情可查看
- [ ] 用户可发起取消申请
- [ ] 管理员可审核取消并正确回退额度

## 5. 数据与运维

- [ ] 数据库连接稳定
- [ ] 错误日志可查看
- [ ] 关键接口失败时前端有可读提示
- [ ] 管理员初始账号和密码已单独保存
- [ ] 本地测试数据与生产数据有隔离方案

## 6. 上线前最后确认

- [ ] 相同版本已经先在测试服验证通过
- [ ] 已按 `docs/16-staging-test-flow.md` 跑完对应的小测试或大测试
- [ ] 移除无用调试信息
- [ ] README 和当前状态文档已同步到网页优先口径
- [ ] 小程序路线已明确暂停，不影响网页主线
- [ ] 已完成一次完整冒烟测试
- [ ] 已整理人工验收步骤

## 7. 建议的人工验收顺序

1. 访问首页，确认商品列表加载。
2. 注册新账号并登录。
3. 管理员后台给测试账号加额度。
4. 用户下单一个有库存的商品。
5. 用户提交取消申请。
6. 管理员审核取消，确认额度回退。
