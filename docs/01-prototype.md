# 01 原型（MVP）

## 页面结构
1) 登录页
- 手机号输入 + 短信验证码

2) 用户端
- 我的信息（只显示本人）
- 我的记录（可新增/编辑必填字段，未审核前可修改）
- 提交后状态：待确认 / 已确认 / 有争议

3) 管理端
- 全部记录总表（筛选：用户/时间/状态/字段）
- 待确认列表（批量确认/驳回）
- 用户管理（禁用/备注）

## 字段清单（来自现有表格）
说明：下列“必填字段”由用户填写；“不填写字段”由管理员填写。

必填字段（用户填写）：
- 游戏名
- 这周宝库是否需要我帮你做
- 收到卷卷
- id（游戏ID / 活动ID）
- 参与大杂鱼炼功（1=我帮你 / 2=自己做）

不填写字段（管理员填写）：
- 我给了
- 绿色
- 蓝色
- 紫色
- 橙色
- 红色
- 金色

选填字段（用户填写）：
- 联系人/微信名

## 交互规则
- 用户仅能编辑“必填/选填字段”
- 管理员仅能编辑“不填写字段”
- 记录确认后锁定，需管理员解锁才能再次修改

## 流水明细（新增）
用于记录每一次“收/发卷、转账、卡片发放”的明细，便于对账追溯。
