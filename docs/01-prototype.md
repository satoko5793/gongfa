# 01 原型（MVP）

## 页面结构

### 前台

1. 首页 `/`
- 商品列表
- 筛选和搜索
- 商品卡片
- 绑定状态与额度入口

2. 商品详情
- 展示卡片图片、名称、属性、标签、价格、库存状态
- 可直接下单

3. 绑定页 `/bind`
- 通过 helper 扫码登录
- 选择角色并绑定账号
- 展示绑定结果

4. 我的页面 `/me`
- 当前绑定角色信息
- 当前额度余额
- 最近订单

5. 我的订单 `/me/orders`
- 订单列表
- 订单状态
- 订单商品快照

### 后台

1. 导入页 `/admin/import`
- 上传 JSON
- 预览解析结果
- 批量生成商品

2. 商品管理页 `/admin/products`
- 编辑商品基础信息
- 上下架
- 修改价格
- 修改库存
- 查看导入来源

3. 用户管理页 `/admin/users`
- 查看已绑定用户
- 调整额度
- 禁用用户

4. 订单管理页 `/admin/orders`
- 查看订单列表
- 确认/取消订单
- 备注处理结果

## 关键交互

### 管理员导入商品

1. 上传 `legacy_getinfo*.json`
2. 系统解析出卡片列表
3. 管理员确认导入
4. 生成商品草稿
5. 补充价格、库存、状态后上架

### 用户绑定与下单

1. 用户进入商城
2. 若未绑定，则前往绑定页
3. 通过 helper 扫码登录并选择角色
4. 完成绑定后返回商城
5. 浏览商品并提交订单
6. 系统校验额度与库存
7. 创建订单并扣减额度

## MVP 商品字段

- 商品名称
- 商品图片
- legacyId
- uId
- 攻击值
- 血量
- 主词条
- 附加词条
- 价格（额度）
- 库存
- 状态（草稿/上架/下架/售罄）

## MVP 订单规则

- 只允许已绑定用户下单
- 只允许购买已上架且库存充足的商品
- 暂不支持购物车，先做单商品直接下单
- 下单成功后立即扣减额度
- 管理员可手工确认或取消订单

## 后续优化方向

- 购物车与多商品订单
- 更细的标签和筛选
- 发货备注和履约状态
- 用户充值申请与审核
