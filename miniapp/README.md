# 小程序前台工作区

这个目录单独用于“小程序前台”开发，不影响当前网页端主线。

当前目标：

- 先保留网页后台
- 单独推进小程序用户端
- 后端优先复用现有商城 API
- 后续再决定是否迁到微信云托管或 CloudBase
- 用户体系走最简方案：`游戏 ID + 游戏名称 + 密码`
- 不接 helper

## 当前目录说明

- `app.js` / `app.json` / `app.wxss`
  - 原生微信小程序最小骨架
- `pages/index/`
  - 商品列表页骨架
- `pages/account/`
  - 账号页骨架
- `utils/config.js`
  - 小程序接口地址配置
- `utils/request.js`
  - 小程序请求封装
- `MIGRATION.md`
  - 从现有网页端迁移到小程序的拆解说明
- `PLAN.md`
  - 小程序详细实施计划和阶段记录

## 当前阶段

现在只是先把工作区拆出来，还没有切到正式小程序开发。

也就是说：

- 当前网页端仍然是主线
- 小程序目录用于后续单独推进

## 下一步建议

建议按这个顺序继续：

1. 先补商品列表和商品详情
2. 再补账号、额度和订单
3. 再补购买闭环
4. 最后决定是否接云托管 / CloudBase
