# 小程序迁移说明

## 当前结论

当前系统拆成两部分最稳：

- 网页后台继续保留
- 小程序只负责用户端

小程序用户端当前统一走最简方案：

- 游戏 ID
- 游戏名称
- 密码

不接 helper。

这样可以最大化复用现有逻辑：

- 商品
- 定价
- 额度
- 订单
- 管理后台

## 现有项目可复用部分

- `backend/`
  - 商品接口
  - 用户接口
  - 订单接口
  - 定价逻辑
- `docs/`
  - 数据模型
  - API 设计
  - 定价规则

## 小程序要重做的部分

- 商品列表页
- 商品详情页
- 登录页
- 我的账户页
- 我的订单页

## 推荐阶段

### 阶段 1

先接只读能力：

- 商品列表
- 商品详情

### 阶段 2

再接用户能力：

- 登录
- 我的额度
- 我的订单

### 阶段 3

再接购买能力：

- 下单确认
- 库存变化
- 订单状态刷新

## 部署路线

推荐优先级：

1. 现有后端先上微信云托管
2. 小程序前台直接调云托管接口
3. 后续再看是否迁到 CloudBase

## 当前不做

- 不接微信支付
- 不接小程序审核适配
- 不重写后台
- 不立即替换网页端
- 不接 helper
